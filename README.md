# Java 学习答疑 Agent

> 基于大语言模型的 Java 学习答疑工具，解决编程初学者的核心痛点

## 项目简介

本项目构建了一个基于大语言模型的「Java 学习答疑 Agent」，旨在解决大一编程初学者在学习 Java 过程中遇到的两大核心痛点：

1. **针对性不足** - 遇到代码报错、概念理解偏差时，无法得到针对性的、贴合新手视角的分步讲解
2. **知识零散** - 零散的知识点难以串联，缺乏系统化的学习路径规划

## 核心功能

- **意图识别**：自动判断问题类型（语法报错 / 概念理解 / 作业答疑）
- **知识库匹配**：结合 Java 基础知识进行上下文匹配
- **反思校验**：对生成的答案进行二次校验，避免超纲内容或复杂术语堆砌
- **结构化输出**：「问题拆解 + 分步讲解 + 示例代码 + 易错点提醒」四段式闭环

## 技术架构

`
用户问题 → 意图识别 → 知识库匹配 → 反思校验 → 结构化输出
  │           │            │           │
  └───────────┴────────────┴───────────┴── 单 Agent 长链推理流程
`

### 核心模块

| 模块 | 职责 |
|------|------|
| IntentRecognizer | 意图识别，判断问题类型 |
| KnowledgeBase | 本地知识库，存储 Java 基础知识 |
| ReflectionModule | 反思校验，过滤超纲术语 |
| AnswerGenerator | 答案生成，四段式结构输出 |
| JavaLearningAgent | Agent 核心编排 |

## 效果数据

| 指标 | 数值 |
|------|------|
| 单次交互 Token 消耗 | 约 200-500 tokens |
| 日均 Token 消耗 | 约 3-5 万 tokens |
| 学习效率提升 | 约 60% |

相比直接在通用 AI 平台提问，通过问题分类和上下文截断策略，显著降低了无效 Token 的浪费。

## 技术栈

- **语言**：Java 17
- **构建**：Maven
- **依赖**：Gson（JSON 处理）

## 快速开始

### 环境要求

- JDK 17+
- Maven 3.6+

### 编译运行

`ash
cd java-learning-agent
mvn clean package
java -jar target/java-learning-agent-1.0.0.jar
`

## 设计理念

1. **新手友好**：过滤复杂术语，用通俗语言解释概念
2. **成本可控**：通过意图分类减少无效 Token 消耗
3. **结构清晰**：四段式输出，让学习路径可追踪
